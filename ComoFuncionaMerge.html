Como funciona o git merge

O comando git merge é usado para unir o histórico de dois branches diferentes. Normalmente, você o utiliza para trazer as alterações de um branch secundário para o branch principal.

Passos básicos

  • git checkout main – Primeiro, vá para o branch onde deseja aplicar as mudanças (ex: main).
  • git merge nome-do-branch – Faz a fusão do branch especificado com o branch atual.

Tipos de Merge

  • Fast-forward: acontece quando o branch atual pode simplesmente avançar para o commit do outro branch, sem criar um novo commit de merge.
  • Merge commit: cria um novo commit especial que une os históricos dos dois branches.

Resolvendo Conflitos

Às vezes, o Git não consegue decidir automaticamente quais mudanças manter. Isso gera um conflito. Nesse caso:

  • Edite os arquivos conflitantes e escolha quais alterações manter.
  • Depois, use git add <arquivo> para marcar como resolvido.
  • Finalize com git commit para concluir o merge.

Exemplo prático

# Estando no branch main
git checkout main
git merge feature-login

Esse comando trará todas as alterações do branch feature-login para o main.
